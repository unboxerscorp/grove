# grove v1.9+ 브레인스토밍

> Status: historical v1 brainstorm; superseded by the current v2 live cockpit model documented in `docs/design/MASTER_NODE.md` and `docs/agents/LEAD-HANDOFF.md`.
> 작성일: 2026-06-04

## 전제

- v1.0~v1.7은 출시 완료 상태로 본다. v1.7은 decision inbox와 board event cursor replay로 사람 판단과 live board 정합성을 강화했다.
- v1.8은 presence와 notification rules를 진행 중인 것으로 둔다.
- v1.9는 협업 알림 기반 위에서 "재현 가능한 팀룸, 안전한 자율 실행, dashboard 밖 액션"을 묶는 단계가 적합하다.
- 장기 모델은 로컬-퍼-멤버 실행, 실제 CLI 세션, 보드=위임 프로토콜, Tailscale 기반 선택 공유, 사람이 최종 판단하는 운영 흐름이다.

## 우선순위 기준

- **P0**: v1.9 핵심 후보. v1.8 presence/notification 기반을 바로 활용하고 제품 완성도가 크게 오른다.
- **P1**: v1.9 stretch 또는 v2.0 후보. 가치가 크지만 권한, 신뢰, 데이터 품질 의존성이 있다.
- **P2**: v2.0+ 후보. 멀티머신 실행·handoff처럼 프로토콜 설계가 커진다.

## 2. 제안 v1.9 스코프

v1.9의 권장 테마는 **"portable autonomy and action surfaces"**다. v1.8에서 presence와 notification rules가 들어오면, v1.9는 팀룸을 복제 가능하게 만들고, node가 안전한 범위에서 스스로 일감을 잡게 하며, 사람이 dashboard 밖에서도 병목을 풀 수 있게 하는 것이 자연스럽다.

### v1.9 핵심 항목

1. **온보딩 위저드 v2 + project import/export + template marketplace v1**
   - 새 멤버가 template에서 room을 만들고, 기존 room을 redacted bundle로 내보내고, 검증된 template를 설치한다.
   - local-first marketplace로 시작하고, 네트워크 배포나 public registry는 후속으로 둔다.

2. **self-retro lane v1 + review sampling policy**
   - daily/weekly briefing, 개선 task 생성, risky completion review 위임을 묶어 팀 운영 학습 루프를 만든다.
   - 결과는 board comment/task로 남겨 사람이 검토 가능해야 한다.

3. **guarded autonomous pickup v0 + routing planner v0**
   - 자동 claim은 opt-in node, healthy/recoverable 상태, WIP budget, role drift guard 통과 시에만 허용한다.
   - routing planner는 자동 위임이 아니라 추천과 근거 표시로 시작한다.

4. **Slack command surface v0 + mobile actions v1**
   - unblock, approve, comment, status, assign 같은 작은 운영 액션을 dashboard 밖에서 처리한다.
   - delegate와 destructive action은 role gate와 replay guard가 충분한 경우에만 허용한다.

5. **multi-machine summary aggregation v0 as stretch**
   - 실행 handoff가 아니라 read-only status summary부터 시작한다.
   - signed summary와 privacy policy가 준비되지 않으면 v2.0으로 넘긴다.

### v1.9 exit criteria

1. onboarding wizard가 template 선택, 환경 점검, 첫 smoke task 완료까지 안내한다.
2. project import/export가 org, roles, board snapshot, template metadata를 redacted bundle로 round-trip한다.
3. template marketplace가 local template preview, install, update, smoke check를 제공한다.
4. self-retro lane이 daily/weekly briefing과 최소 1개의 개선 task를 만들고 근거 event를 링크한다.
5. review sampling policy가 risky completion을 reviewer lane에 위임하고 결과를 원 task에 comment로 연결한다.
6. guarded autonomous pickup이 opt-in node에서만 claim하고 unsafe 상태에서는 제안 comment로 fallback한다.
7. routing planner가 추천 assignee, confidence, 근거를 보여주되 자동 위임은 하지 않는다.
8. Slack command와 mobile action이 comment/unblock/approve/status/assign 중 최소 4개를 role-gated로 처리한다.
9. multi-machine summary aggregation은 read-only prototype 또는 명확한 v2.0 설계 문서로 마감한다.
10. 실제 서버 e2e가 import/export, marketplace, self-retro, guarded pickup, Slack/mobile action의 핵심 계약을 검증한다.

## 3. v2.0+ 백로그

| 후보                                   | 설명                                                                    | 넘기는 이유                                                                |
| -------------------------------------- | ----------------------------------------------------------------------- | -------------------------------------------------------------------------- |
| multi-machine summary aggregation full | 여러 로컬 room 상태를 한 web에서 안정적으로 모은다.                     | project identity, signed summary, privacy/redaction policy가 더 필요하다.  |

| cost-aware planner full                | 비용/credit을 반영해 assignee 추천과 실행 예산을 관리한다.              | cost source 신뢰도와 예산 정책을 더 쌓아야 한다.                           |

| Slack command full destructive set     | spawn/despawn, secret update, cross-room handoff까지 채널에서 처리한다. | replay, spoofing, approval, audit UX 위험이 크다.                          |

## 4. 실행 순서 제안

1. **V9-W1 portability foundation**: import/export schema, redaction, template manifest, marketplace preview/install.
2. **V9-W2 onboarding**: wizard v2, environment checks, template smoke task, support bundle path.
3. **V9-W3 self-retro and review**: briefing generator, improvement tasks, risky completion sampling.
4. **V9-W4 guarded autonomy**: autonomous pickup v0, WIP/backpressure guard, role drift detector, routing planner.
5. **V9-W5 action surfaces**: Slack command v0, mobile action v1, replay/role/audit guards.
6. **V9-W6 stretch and hardening**: read-only multi-machine summary prototype or design, real-server e2e, docs/changelog.

## 5. 주요 리스크

- import/export와 template marketplace는 secret redaction을 실수하면 제품 신뢰를 잃는다. 기본 export는 보수적으로 작게 시작해야 한다.
- autonomous pickup은 반드시 visible CLI session, board/audit 기록, opt-in, backpressure, unsafe fallback을 갖춰야 한다.
- Slack command와 mobile action은 편하지만 spoofing/replay/권한 혼동이 크다. 작은 command set부터 role-gated로 검증한다.
- multi-machine은 제품 비전의 큰 축이지만 v1.9 core로 넣기엔 신뢰 모델이 크다. read-only summary부터 시작하는 것이 현실적이다.
