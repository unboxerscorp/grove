# grove v1.8+ 브레인스토밍

작성일: 2026-06-04

## 전제

- v1.0~v1.6은 출시 완료 상태로 본다. v1.6은 node auto-recovery와 delegation-chain explorer로 24/7 운영과 위임 가시성을 강화했다.
- v1.7은 decision inbox, presence, board cursor replay를 진행 중인 것으로 둔다.
- v1.8은 "협업 가능한 팀룸"을 넘어 "스스로 운영 신호를 만들고, 새 멤버가 재현하고, 제한된 자율 실행까지 믿고 맡길 수 있는 팀룸"을 목표로 한다.
- 장기 모델은 로컬-퍼-멤버 실행, Tailscale 공유, 실제 CLI 세션 실행, 보드=위임 프로토콜, 사람 판단 지점의 명확한 inbox다.

## 우선순위 기준

- **P0**: v1.8 핵심 후보. v1.7 기반 위에서 바로 구현 가치가 크고 운영 리스크가 낮다.
- **P1**: v1.8 stretch 또는 v1.9 후보. 가치가 크지만 정책, 데이터 품질, 신뢰 모델이 더 필요하다.
- **P2**: v1.9+ 후보. 멀티머신, 자동 라우팅, 비용 최적화처럼 제품 축을 넓힌다.

## 2. 제안 v1.8 스코프

v1.8의 권장 테마는 **"operational autonomy, safely"**다. v1.7이 사람이 보고 판단하는 협업 surface를 닫으면, v1.8은 알림·회고·template·제한된 자율 pickup을 묶어 room이 스스로 운영 신호를 만들게 한다. 다만 멀티머신과 비용 기반 자동 라우팅은 신뢰 모델과 데이터 품질이 더 필요하므로 core가 아니라 stretch/backlog로 둔다.

### v1.8 핵심 항목

1. **notification rules v1 + preference center**
   - decision inbox를 실제 운영 알림 원장으로 만든다.
   - severity, 멤버 preference, quiet hours, Slack/digest target까지 포함한다.

2. **온보딩 위저드 v2 + project import/export + support bundle**
   - 새 멤버가 template에서 room을 만들고, 기존 room을 redacted bundle로 공유/재현한다.
   - export와 support bundle을 분리해 제품 재사용과 문제 재현을 모두 지원한다.

3. **self-retro lane v1 + daily briefing**
   - board/audit/recovery/notification/cost unknown을 요약해 개선 task와 daily briefing을 만든다.
   - 자동 실행보다 사람이 검토 가능한 제안과 위임 가능한 task로 시작한다.

4. **template marketplace seed + template test harness**
   - local template preview/install/smoke check를 닫아 로컬-퍼-멤버 모델의 재사용성을 강화한다.
   - marketplace는 네트워크 제품이 아니라 "검증 가능한 template source"부터 시작한다.

5. **autonomous pickup v0 guarded**
   - 특정 node가 자기 ready task를 제한적으로 claim하거나 claim 제안을 남긴다.
   - backpressure, recovery status, role drift detector, notification rule이 모두 통과할 때만 자동 claim을 허용한다.

6. **routing recommender v0**
   - 자동 위임이 아니라 assignee 추천으로 시작한다.
   - WIP, 최근 실패, role, delegation chain, cost confidence를 근거와 함께 보여준다.

### v1.8 exit criteria

1. notification rules가 blocked aging, stale claim, dead/stale node, mention, delegated-to-me, cost unknown 중 최소 6종을 처리한다.
2. member별 preference로 Slack/digest/dashboard target과 quiet hours를 설정할 수 있다.
3. onboarding wizard가 template 선택부터 smoke task 완료까지 안내한다.
4. project import/export가 org, roles, board snapshot, template metadata를 redacted bundle로 round-trip한다.
5. support bundle이 health, audit slice, board slice, recovery events를 secret/path redaction과 함께 생성한다.
6. self-retro lane이 daily briefing과 최소 1개의 개선 task를 생성하고 원인 event 링크를 남긴다.
7. template seed flow가 local template preview, install, template smoke check를 통과한다.
8. autonomous pickup v0가 opt-in node에서만 동작하고, unsafe 상태에서는 claim 대신 제안 comment를 남긴다.
9. routing recommender가 추천 assignee와 근거를 제공하되 자동 위임은 하지 않는다.
10. 실제 서버 e2e가 notification, export/import, support bundle, self-retro, guarded pickup의 핵심 계약을 검증한다.

## 4. 실행 순서 제안

1. **V8-W1 notification engine**: rule store, preference center, decision inbox 연결, digest target.
2. **V8-W2 portability**: project export/import schema, redaction policy, support bundle, round-trip tests.
3. **V8-W3 onboarding and templates**: onboarding wizard v2, template seed install/preview, template smoke harness.
4. **V8-W4 self-retro**: daily briefing, improvement task generation, event links, reviewer handoff.
5. **V8-W5 guarded autonomy**: autonomous pickup v0, role drift detector, backpressure guard, routing recommender v0.
6. **V8-W6 hardening**: real-server e2e, notification chaos cases, export redaction review, docs/changelog.

## 5. 주요 리스크

- notification rules는 빠르게 noise가 된다. preference center와 severity model을 같은 release에 넣어야 한다.
- import/export와 support bundle은 secret/path redaction 실패가 가장 큰 위험이다. 기본값은 과하게 보수적으로 두고 사용자가 opt-in으로 더 넣게 한다.
- autonomous pickup은 제품 비전과 맞지만 잘못 들어가면 사용자가 모르는 실행처럼 느껴질 수 있다. 반드시 visible CLI 세션, opt-in node, board/audit 기록, unsafe 시 제안 comment라는 guard를 둔다.
- routing recommender와 cost-aware planner는 근거 품질이 낮으면 신뢰를 잃는다. v1.8은 추천과 근거 표시까지, 비용 기반 자동 결정은 v1.9+가 맞다.
