# grove v1.7+ 브레인스토밍

> Status: historical v1 brainstorm; superseded by the current v2 live cockpit model documented in `docs/design/MASTER_NODE.md` and `docs/agents/LEAD-HANDOFF.md`.
> 작성일: 2026-06-04

## 전제

- v1.0~v1.5는 출시 완료 상태로 본다. v1.5는 실제 서버 e2e, dashboard delegate, backend 기반 node-status polish를 닫았다.
- v1.6은 auto-recovery, board event cursor replay, delegation-chain explorer를 중심으로 진행 중이다.
- v1.7은 v1.6의 운영 안정화 위에 "여러 사람이 같이 보고, 빠르게 시작하고, 놓치지 않고, 팀 방식이 축적되는" 제품 단계를 목표로 한다.
- 장기 비전은 보이는 실제 CLI 세션, 로컬-퍼-멤버 실행, Tailscale 기반 선택 공유, 보드=위임 프로토콜, 인간 판단 지점의 명확한 surface다.

## 우선순위 기준

- **P0**: v1.7 핵심 후보. v1.6 안정화 이후 바로 사용자 가치가 크고, 기반 의존성이 충분하다.
- **P1**: v1.7 stretch 또는 v1.8 후보. 가치가 크지만 정책·보안·데이터 품질 의존이 더 있다.
- **P2**: v1.8+ backlog. 큰 설계나 cross-machine 신뢰 모델이 필요하다.

## 3. 제안 v1.7 스코프

v1.7의 권장 테마는 **"협업 가능한 portable team room"**이다. v1.6이 복구, cursor replay, 위임 체인을 안정화하면 v1.7은 여러 사람이 같은 room을 보며 판단하고, 새 멤버가 같은 room을 복제하며, 팀 운영이 자동으로 회고되는 방향이 가장 자연스럽다.

### v1.7 핵심 항목

1. **presence + notification rules + decision inbox**
   - member presence, view/edit cursor, notification rules, decision inbox를 묶어 "사람이 필요한 일"을 놓치지 않게 한다.
   - Slack은 v1.7 core에서는 알림과 thread reply/comment/unblock까지로 제한하고, 임의 command surface는 stretch로 둔다.

2. **온보딩 위저드 v2 + project import/export MVP**
   - 로컬-퍼-멤버 모델의 반복 설치, template 시작, 지원 재현을 해결한다.
   - export 기본값은 redacted bundle이며, secret과 credential은 별도 안내만 제공한다.

3. **self-retro lane v1 + daily briefing**
   - audit, board, recovery, cost unknown, notification history를 요약해 개선 task와 daily briefing을 만든다.
   - AI 활용은 자동 실행보다 "보이는 제안과 위임 가능한 개선 task"로 시작한다.

4. **template marketplace seed**
   - marketplace 전체 제품을 만들기보다 local folder/repo template install, preview, smoke check까지만 닫는다.
   - import/export MVP가 안정되면 바로 얹기 좋다.

5. **review sampling policy v0**
   - completed task 중 위험도가 높은 작업을 reviewer lane에 자동 위임한다.
   - self-retro와 별개로 품질 안전망을 실제 board workflow에 연결한다.

### v1.7 exit criteria

1. 두 명 이상의 member가 같은 dashboard에 접속하면 presence와 현재 보는 surface가 표시된다.
2. blocked aging, dead/stale node, stale claim, mention, delegated-to-me, cost unknown 중 최소 5종 이벤트가 notification rules로 동작한다.
3. decision inbox에서 human-needed item을 comment/unblock/assign 중 하나로 처리할 수 있고 audit actor가 남는다.
4. onboarding wizard가 새 project를 template에서 만들고 smoke task까지 완료 상태로 안내한다.
5. project export/import가 org, role docs, board snapshot, template metadata를 redacted bundle로 round-trip한다.
6. self-retro lane이 최근 audit/board를 읽어 daily briefing과 최소 1개의 개선 task를 생성한다.
7. template seed flow가 local template preview, install, smoke check를 제공한다.
8. review sampling policy가 위험 task를 reviewer에게 위임하고 결과를 원 task에 comment로 연결한다.
9. 실제 서버 e2e가 presence, notifications, import/export, decision inbox의 핵심 계약을 검증한다.

| -------------------------- | -------------------------------------------------------- | ------------------------------------------------------------------------------ |

| Slack command surface full | delegate/assign/status/unblock을 채널 명령으로 실행한다. | command verification, role policy, replay safety가 커진다.                     |
| mobile view/action         | phone-first read/approval UI를 만든다.                   | presence/notification/decision inbox API가 먼저 안정되어야 한다.               |
| multi-machine aggregation  | 여러 로컬 room summary를 읽기 전용으로 모은다.           | signed summary, project identity, privacy policy가 필요하다.                   |

## 5. 실행 순서 제안

1. **V7-W1 collaboration primitives**: presence model, WS payload, member preference, notification rule store.
2. **V7-W2 decision inbox**: human-needed aggregation, action buttons, audit linkage, Slack reply/comment/unblock 연결.
3. **V7-W3 portability**: project export/import schema, redaction, support bundle, round-trip tests.
4. **V7-W4 onboarding**: wizard UI, environment checks, template install, first smoke task.
5. **V7-W5 self-retro and review sampling**: daily briefing, improvement task generation, risky completion sampling.
6. **V7-W6 template seed and hardening**: local marketplace seed, template smoke harness, real-server e2e, docs and changelog.

## 6. 주요 리스크

- presence와 notification은 작은 UX처럼 보이지만 member session, WS reconnect, privacy preference가 얽힌다. v1.7 초반에 event contract를 먼저 고정해야 한다.
- import/export는 secret redaction 실수가 가장 큰 위험이다. export 기본값은 보수적으로 두고 support bundle과 project bundle을 분리한다.
- autonomous pickup은 매력적이지만 v1.7 core로 넣으면 예측 불가능한 자동 실행이 늘 수 있다. v1.7에서는 self-retro와 review sampling처럼 "제안/위임" 중심의 AI 루프가 더 안전하다.
- Slack command full surface는 권한, replay, spoofing, command ambiguity가 크다. v1.7은 알림과 thread reply 기반 action으로 제한하고 full command는 v1.8로 넘긴다.
