# grove v1.2+ 브레인스토밍

> Status: historical v1 brainstorm; superseded by the current v2 live cockpit model documented in `docs/design/MASTER_NODE.md` and `docs/agents/LEAD-HANDOFF.md`.
> 작성일: 2026-06-04

## 전제

- grove의 장기 목표는 완성형 24/7 팀-OS이자 개발실 제품이다.
- 모든 실행 주체는 실제로 열어 보고 대화할 수 있는 `codex`, `claude`, `agy` CLI 세션이어야 한다.
- 각 팀원은 자기 머신에서 자기 인증으로 grove를 실행하고, 팀 공유는 템플릿과 프로젝트 파일로 한다.
- 팀 접근은 로컬 또는 Tailscale 같은 사설 네트워크를 기본으로 하되, non-loopback 노출 전에는 실 인증이 필요하다.
- lead/orchestrator는 구현하지 않고 분해, 위임, 확인, 통합만 한다.
- 보드는 위임 프로토콜이다. 부모가 자식에게 일을 주는 공식 경로는 보드 task다.

## 우선순위 기준

- P0: v1.2에서 다루지 않으면 제품 안정성과 사용 모델이 흔들리는 항목
- P1: v1.2 핵심 경험을 강하게 만드는 항목
- P2: v1.2 후보이나 범위가 커서 잘라낼 수 있는 항목
- P3: v1.3+로 보내도 되는 확장 항목

## 제안 v1.2 스코프

v1.2의 주제는 “안정적인 계층형 개발실”로 잡는다. v1.1이 안정화와 테스트 확장을 목표로 한다면, v1.2는 실제 매일 쓰는 팀-OS의 첫 완성 경험을 만든다.

### v1.2 핵심 항목

1. **계층형 org + 보드 위임**
   - `grove delegate` 또는 동일 API로 parent → child task assign을 공식화한다.
   - dashboard에서 org graph와 delegate panel을 제공한다.
   - orchestrator-only audit을 추가해 직접 구현 위반을 보이게 한다.

2. **신뢰성 코어**
   - W3.1 이벤트 기반 turn 감지를 도입한다.
   - `agy` interactive adapter를 parity 수준으로 안정화한다.
   - `grove despawn`과 repair/reconnect 흐름을 제공한다.

3. **Slack + ask-human v1**
   - blocked `needs_human` task가 Slack thread로 나가고, reply가 comment + unblock으로 이어진다.
   - Slack chat thread가 grove session으로 sticky routing된다.
   - dashboard에서 manifest 다운로드, token 저장, 연결 상태를 제공한다.

4. **운영 관측성 v0**
   - `/api/health` depth, structured logs, node heatmap, board metrics를 제공한다.
   - cost/token은 v1.2에서 조사/표시 skeleton까지, 특히 `agy` 크레딧 경고 surface를 만든다.

5. **보안 기본선**
   - non-loopback에서 HTML token 주입을 대체할 real auth gate를 제공한다.
   - CSRF, Origin/Host 검증, secret redaction을 regression test로 묶는다.

6. **온보딩과 템플릿**
   - first-run wizard와 `new-project` template 세트를 제공한다.
   - project export/import를 문서화하고 dashboard에서 최소 UI를 제공한다.

### v1.2 Exit Criteria

1. 새 사용자가 가이드대로 설치 후 wizard에서 `lead + maker + reviewer` 프로젝트를 만들고 첫 task를 완료한다.
2. dashboard에서 sublead가 있는 계층형 org를 보고, parent가 child에게 board task를 assign하고, 결과를 fan-in한다.
3. `codex`, `claude`, `agy` 노드가 모두 spawn, turn detect, rebind, despawn 흐름을 통과한다.
